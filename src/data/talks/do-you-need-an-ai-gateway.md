LLM applications increasingly rely on MCP servers for tools, data, and automation,
but teams often hit real-world snags: trusted server discovery, inconsistent authentication, lack of
built-in security features, unreliability of tool calls, tool pollution, lack of proper observability, etc.
All these issues have a significant impact on the developer experience and make it hard to properly
operationalize MCP for real-world production environments.

In this talk, I frame these pain points from a developer and platform persona angles, and discuss how an
MCP gateway can help address them and provide a solid foundation to leverage MCP at scale in enterprise environments.
